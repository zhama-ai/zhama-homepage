# Mini Chat 浮窗 + 下载中心 + Windows 安装器：Tego OS v3.0.0 桌面端体验三连击

> 让数字分身真正装进每天的工作流——不是开一个新 App，而是「随手呼出、用完就走」。

---

## 桌面端为什么重要

服务端再好，落到企业里最终都是用户每天打交道的那个客户端。v3.0.0 在桌面端做了三件事，目标只有一个：

> **让数字分身像 Spotlight、Raycast 一样，随手呼出、用完就走，不打断主任务。**

下面分别讲这三件事。

---

## 一、Mini Chat 浮窗：常驻置顶的「随手分身」

### 设计目标

数字分身的真实使用场景，绝大多数不是「打开一个 App 专门聊一会」，而是：

- 写文档时，问一句术语；
- 看代码时，问一个 API；
- 处理工单时，呼出 IT 助理；
- 写 Listing 时，让分身查一条竞品评论。

这种场景的关键是：**入口必须比打开主应用更轻**，否则用户根本不会用。

### 五个关键体验

#### 1. 独立的 Tauri 窗口，紧凑、无边框、可置顶

Mini Chat 不是主窗口的子窗口，而是一个完全独立的 Tauri webview。紧凑 + 无边框 + 可置顶，让它在屏幕右下角呆几个小时也不影响主任务。

#### 2. 复用同一连接，对话不分裂

Mini Chat 和主窗口共享**同一个会话连接**。在 Mini Chat 里聊两句、再打开主窗口，会话历史是连续的；反过来一样。

#### 3. 全局快捷键，参考 Spotlight / Raycast 行为

可配置的全局快捷键唤出 / 隐藏。我们的默认绑定参考了 Spotlight / Raycast 的行为模型：

- 唤出 → 直接获取焦点；
- 已显示 → 隐藏；
- ESC → 隐藏（不退出）；
- 配置项允许改快捷键，企业可以全公司统一推送。

#### 4. 跨窗口同步

两个窗口始终展示同一会话。Mini Chat 里输入一条消息、回头打开主窗口看完整内容，不需要刷新、不需要重连。

#### 5. 懒创建 + 复用，第一次呼出才创建 webview

Mini Chat 的 webview 是**懒创建**的——不在应用启动时就拉起来，而是在第一次呼出时创建并保留。这是一项关键优化：

- 应用启动不被 Mini Chat 拖慢；
- 后续呼出毫秒级响应；
- 用户从来不用 Mini Chat 的话，永远不会创建。

> 这套设计让 Mini Chat 成为「真正每天会用」的入口，而不是「装了之后就再没打开过」的 Feature。

## 二、Downloads：统一的下载中心

### 痛点

在 v2.x 时代，企业首次拿到 Tego 环境后要分发到不同终端时，会发现：

- 桌面端、移动端、命令行的安装包散落在多个地方；
- 不同员工拿到的是不同版本，常见「我装的是 0.x，TA 装的是 0.y」；
- 下载链接经常过期，IT 同事疲于答疑。

### v3.0.0 的解法

把所有终端的安装包统一聚合到一个**「Downloads 中心」**：

- 桌面端：macOS / Windows / Linux 安装包；
- 移动端：Android / iOS 应用；
- 命令行：CLI 二进制；
- 一目了然的版本号、更新时间、签名指纹。

企业 IT 拿到环境后，可以把这个 Downloads 链接直接发给全公司，剩下的事让员工自己来。

### 配套能力

- **版本对照**：每个安装包标注当前服务端兼容的最低 / 推荐版本；
- **静默下载**：可配置链接，企业内网批量分发；
- **签名指纹**：方便企业 IT 验签，避免被钓鱼包替换。

---

## 三、Windows NSIS 安装器全面重写

Windows 是 v3.0.0 桌面端体验提升最大的一环。v2.x 的 Windows 安装器在公司内网经常踩到这些坑：

- 杀软误报，安装中断；
- 安装路径权限不对，启动后无法写日志；
- 旧版残留，新装不上去；
- 安装包过大，企业带宽紧张时长。

### v3.0.0 的修复清单

1. **NSIS 安装器全面重写**：从结构上重新组织，剔除旧脚本里的隐藏路径写入；
2. **静默安装**：支持 `/S` 参数，方便 IT 走 SCCM / Intune 批量推送；
3. **自动卸载旧版本**：安装新版前自动检测并卸载旧版，干净不残留；
4. **安装包瘦身**：去掉冗余依赖与调试符号，体积明显减小；
5. **签名兼容性**：兼容主流杀软的签名校验，降低误报率；
6. **可选自定义路径**：企业可以指定标准化的安装目录（C 盘 / D 盘）。

> 「能不能在 Windows 上稳定装上 + 升级」是企业大批量铺开 Tego 桌面端的硬门槛。v3.0.0 把这条门槛往下压了不止一档。

## 三件事是一个整体

Mini Chat、Downloads、Windows 安装器——表面看是三个不相关的功能，但落在「让数字分身真正进入每天的工作流」这件事上，它们是一个整体：

- **Windows 安装器** → 装得上；
- **Downloads 中心** → 装得齐、装得对版本；
- **Mini Chat** → 装上之后，每天会用。

少了任何一环，桌面端体验都会出现明显短板。

---

## 与服务端能力的联动

桌面端不只是「客户端体验」——它和 v3.0.0 的服务端能力深度联动：

- Mini Chat 和主窗口的会话同步，依赖 v3.0.0 的统一会话契约；
- 桌面端走 `per_user` 时，每个 Windows 账户对应一个独立运行时（不与同事串扰）；
- 桌面端访问技能、记忆、模板时，所有读写经过 v3.0.0 的网关 + 三层 fallback；
- 桌面端的所有交互行为，都会被聚合到 BusinessMonitor 的「参与」面板。

桌面端是 v3.0.0 端到端体验的「最后一公里」。

## 如何上手

1. 从 Downloads 中心拉到对应平台的安装包；
2. （Windows）确认企业 IT 完成签名信任配置；
3. 安装并登录企业账号；
4. 配置全局快捷键，唤出 Mini Chat；
5. 把 Mini Chat 拖到屏幕角落，开始你的一天。
